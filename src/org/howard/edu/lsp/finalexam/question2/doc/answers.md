# Question 2 – Design Explanation

The Template Method pattern is implemented in the abstract `Report` class. The `generateReport()` method is declared `final` and defines the fixed workflow: it calls `loadData()`, then `formatHeader()`, `formatBody()`, and `formatFooter()` in that exact order. These four steps are declared `abstract`, so each concrete subclass — `StudentReport` and `CourseReport` — must supply its own implementations. The invariant part (the sequence and the section labels like `=== HEADER ===`) lives in the base class and can never be changed by subclasses, while the variant part (what actually appears in each section) is fully delegated to the subclass. The `Driver` demonstrates polymorphism by iterating a `List<Report>` and calling `generateReport()` on each element without knowing the concrete type.
